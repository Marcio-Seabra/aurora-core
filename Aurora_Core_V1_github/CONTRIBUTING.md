# Contribuindo

Obrigado por contribuir.

## Fluxo
1. Crie uma branch com um nome claro.
2. Descreva o problema e a solucao no commit.
3. Garanta que o projeto roda em Python 3.11.

## Boas praticas
- Evite adicionar dados pessoais em `memory/` ou `data/`.
- Mantenha funcoes pequenas e testaveis.
- Documente mudancas no README quando afetarem uso.
