========================================
AURORA – ARQUITETURA LÓGICA DO SISTEMA
Projeto: Eclipse Archives
========================================
Requisito: Python 3.11

Este documento define a organização, a função e o fluxo de dados
de todas as pastas, arquivos e scripts utilizados pela IA Aurora.

Aurora é uma IA núcleo:
- NÃO executa ações
- NÃO acessa arquivos diretamente
- NÃO mantém estado interno persistente
- SOMENTE lê contexto preparado externamente

----------------------------------------
VISÃO GERAL DO FLUXO
----------------------------------------

Entrada bruta (texto)
→ Memória de curto prazo
→ Resumo
→ Memória de longo prazo
→ Contexto preparado
→ Aurora (LLM)

----------------------------------------
ESTRUTURA DE PASTAS
----------------------------------------

/core
/context
/data
/decision_layer
/memory
/logs
/scripts
/services

----------------------------------------
PASTA: core/
----------------------------------------

Função:
- Contém o núcleo cognitivo da Aurora
- Define comportamento, tom, formato de resposta e limites

Conteúdo típico:
- aurora_core.py (opcional, se houver encapsulamento)
- Definição de contrato de resposta (fala, intenção, estado, observações)

Acesso:
- Lê: contextos prontos (via chamada externa)
- Escreve: NUNCA

----------------------------------------
PASTA: context/
----------------------------------------

Função:
- Contexto montado para envio à Aurora
- Janela de contexto temporária

Conteúdo típico:
- context.json ou context.txt
- Conteúdo já filtrado, resumido e seguro

Acesso:
- Lê: decision_layer
- Escreve: services (context builder)

----------------------------------------
PASTA: data/
----------------------------------------

Função:
- Entrada bruta do sistema
- Dados não processados

Conteúdo típico:
- conversation_raw.txt
- dumps de logs externos
- entrada de sensores ou interfaces

Acesso:
- Lê: scripts
- Escreve: sistemas externos (chat, API, usuário)

----------------------------------------
PASTA: memory/
----------------------------------------

Função:
- Memória persistente da Aurora
- Nunca acessada diretamente pela LLM

Subpastas:

--- memory/identity/
Função:
- Identidade fixa da Aurora
- Informações do criador e do projeto
- Regras permanentes

Exemplos:
- creator.json
- aurora_identity.json

Características:
- Escrita manual ou administrativa
- Alteração rara

--- memory/short_term/
Função:
- Memória de curto prazo
- Conversas recentes
- Contexto descartável

Exemplos:
- session_20260125_001.json

Características:
- Escrita automática
- Limpeza periódica

--- memory/summaries/
Função:
- Resumos de sessões
- Compressão de contexto

Exemplos:
- summary_session_001.json

Características:
- Criada a partir de short_term
- Base para promoção a longo prazo

--- memory/long_term/
Função:
- Conhecimento persistente
- Fatos duradouros
- Decisões consolidadas

Exemplos:
- knowledge_core.json
- decision_history.json

Características:
- Escrita controlada
- Leitura frequente

----------------------------------------
PASTA: decision_layer/
----------------------------------------

Função:
- Camada de decisão externa à Aurora
- Orquestra o sistema

Responsabilidades:
- Escolher qual memória usar
- Decidir quando resumir
- Decidir quando promover memória
- Preparar contexto

Acesso:
- Lê: memory/*
- Escreve: context/

----------------------------------------
PASTA: services/
----------------------------------------

Função:
- Lógica reutilizável
- Não executável diretamente

Scripts típicos:
- memory_parser.py
- memory_classifier.py
- memory_writer.py
- summary_builder.py
- context_builder.py

Características:
- Apenas funções
- Sem efeitos colaterais diretos
- Usados por scripts

----------------------------------------
PASTA: scripts/
----------------------------------------

Função:
- Pontos de execução
- Orquestram o sistema

Scripts típicos:
- ingest_memory.py
- build_context.py
- promote_memory.py
- cleanup_short_term.py

Características:
- Podem ser executados via terminal
- Usam services
- Nunca contêm lógica pesada

----------------------------------------
PASTA: logs/
----------------------------------------

Função:
- Auditoria
- Rastreamento de eventos

Conteúdo típico:
- memory.log
- decision.log
- system.log

Acesso:
- Escrita: scripts e services
- Leitura: humana / debug

----------------------------------------
RELACIONAMENTO COM A AURORA (LLM)
----------------------------------------

Aurora:
- NÃO sabe que existem arquivos
- NÃO sabe que existem pastas
- NÃO mantém memória própria

Ela recebe apenas:
- Texto
- Contexto preparado
- Regras de comportamento

Toda memória é EXTERNA.

----------------------------------------
PRINCÍPIO FUNDAMENTAL
----------------------------------------

Aurora pensa.
O sistema lembra.
O sistema decide quando lembrar.
Aurora apenas responde.
